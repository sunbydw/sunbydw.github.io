Notes of Binary Search

Posted on 2016-12-30 | In Algorithm

相关变量： mid, lo, hi

首先看题目设定，看二分缩小范围后midd是包括在右半部分还是左半部分，还是都不包括，据此衍生出三种模式：

  1. 中间值包括在左半部分

  2. 中间值包括在右半部分

    记忆方法： 这两种形式的区别相当于偶数个数的时候取左边的中位数还是右边的中位数。中间值包括在右边，这样取右边的中位数，就两边更平衡一些

  3. 中间值两边都不包括

前两种模式结束后都是 lo == high，因此返回哪个都行；后一种结束为：lo == high + 1，但是一般都返回false了（二分查找没找到）

例子：

  1. 中间值包括在左半部分：

    Suppose a sorted array is rotated at some pivot unknown to you beforehand.

    (i.e., 0 1 2 4 5 6 7 might become 4 5 6 7 0 1 2).

    Find the minimum element. You may assume no duplicate exists in the array.

    以[4,5,6,7,0,1,2] 为例，第一次肯定是走右边（7 > 2), 7肯定不会是最小的，所以 lo 取7 下一个位置；若是[5,6,7,0,1,2,4], 第一次走左边，0是最小，肯定要包括。 所以中间值包括在左半部分

    ​

  2. 中间值包括在右半部分：

    Given n pieces of wood with length L[i] (integer array).
    Cut them into small pieces to guarantee you could have equal or more than k pieces with the same length. What is the longest length you can get from the n pieces of wood? Given L & k, return the maximum length of the small pieces.

    Example
    For L=[232, 124, 456], k=7, return 114.

​ 当一个长度长了，导致k不够，这时需要减小长度（走左边），这个太长的长度本身不能作为答案；当一个长度短了，导致k大了或刚好等于，这时增加长度可能有更好的答案，也可能没有，因此我们需要把这个长度包含在下次的迭代里的。所以这种情况下中间值包括在右半部分。

  1. 中间值两边都不包括

    Given a sorted array and a target value, return the index if the target is found. If not, return the index where it would be if it were inserted in order.

    You may assume NO duplicates in the array.

    这道题相当于二分查找。二分查找中，无论走左边还是走右边，中间值都是不符合要求的，因此可以直接剔除。

Optimal BSTs

Posted on 2016-12-26 | In Algorithms

Setup:

Input: Frequencies $p_1, p_2,…, p_n$ for items 1,2,…,n

Goal: Compute a valid search tree that minimizes the weighted (average) search time.

$$C(T) = \sum_{items} p_i * (depth(i)+1)$$

Analysis:

Optimal Substructure

Suppose an optimal BST for keys {1, 2, . . . , n} has root r , left subtree $T_1$ , right subtree $T_2$. Then $T_1$ is optimal for the keys {1, 2, . . . , r − 1} and $T_2$ for the keys {r + 1, r + 2, . . . , n} (contradiction)

At this time, we have:

$$C(T) = \sum_{items} p_i + C(T_1) + C(T_2)$$

Relevant Subproblems

Contiguous intervals (S = {i, i + 1, . . . , j − 1, j} for every i ≤ j) Note: Since we sometimes have to calculate the suffix of a prefix(this is not neither a prefix or suffix, but an contiguous interval)

The Recurrence:

Notation: For 1 ≤ i ≤ j ≤ n, let $C_ij$ = weighted search cost of an optimal BST for the items { i, i + 1, . . . , j−1, j } [with probabilities $[ p_i , p_{i+1} , . . . , p_j ]$

Recurrence: For every 1 ≤ i ≤ j ≤ n: $$C_{ij}=min_{r=1,…,j}{\sum^j_{k=i}p_k + C_{i,r-1}+C_{r+1,j}}$$

Algorithm:

Important: Solve smallest subproblems first

Complexity Analysis:

$-\Theta(n^2)$ Subproblems

$-\Theta(j-i)$ time to compute A[i][j]

$-\Theta(n^3)​$ time overall

    For a optimal alignment of X, Y, there are 3 relevant possibilities for the contents of the final position:

Principles of Dynamic Programming

Posted on 2016-12-26 | In Algorithms

Principles of Dynamic Programming:

  1. Identify a small number of subproblems.
  2. Can quickly+correctly solve “larger” subproblems given the solutions to “smaller subproblems”.
  3. After solving all subproblems, can quickly compute the final solution.

Notes:

  1. Subproblems are always some prefix or suffix of the problem.
  2. A form of memorization can not only avoid redundantly calculate the solutions of subproblems, but also help reconstruct specific answer sometimes (not just some kind of value).
  3. 面试中常见的动态规划类型：
    1. 坐标型动态规划 15% 给了一个初始状态和末状态，一般要开初始状态到末状态维度的数组
    2. 序列型动态规划 30% 一个序列，需要开m大小数组
    3. 双序列型动态规划 30% 两个序列，需要开m*n大小的数组
    4. 划分型动态规划 10%
    5. 背包型动态规划 10%
    6. 区间型动态规划 5%
  4. 解题步骤：
    1. 找state(subproblem)
    2. 找function(recurrence)
    3. initialize
    4. 找answer （一般是最后一个数）

Max-Weight Independent Set

Posted on 2016-12-26 | In Algorithms

1. Setup

  • Input: a path graph G = (V, E) with nonnegative weights on vertices.
  • Output: Subset of nonadjacent vertices – an independent set – of maximum total weight.

2. Analysis

Step 1: Formulate Recurrence

  • Reasoning about structure of an optimal solution (narrows down the set of candidates for the optimal solution)
    Let $S \subset V$ be a max-weight independent set (IS). Let $v_n$ = last vertex of path

    • case 1: $v_n\notin S$. Let G’ = G with $v_n$ deleted. S is not only an IS of G’, but also a max-weight IS. (contradiction)
    • case 2: $v_n\in S$. According to the definition of IS: $v_n-1\notin S$. Let G’’ = G with $v_n-1$ deleted. Now $S-{v_n}$ is an max-weight IS of G’’. (contradiction)

  • The strategy here is to compare results derived from two cases and then use the better one to be the current max-weight IS. So current max-weight IS can be represented by previous IS:

    $$IS(G) = max(IS(G’), IS(G’’) + IS)$$

Step 2: Identify the Subproblems

  • IS(G’) and IS(G’’)

Step 3: Use Recurrence from Step 1 to systematically solve all problems

  • Let A[i] be the optimal solution of the vertex set end with vertex i
  • A[] plays a role of memorization, to avoid redundantly calculate the solutions of subproblem.

Knapsack Problem

1. Setup

  • Input:
    • n items. Each has attibutes:
      • value $v_i$(nonnegative)
      • size $w_i$(nonnegative and integral)
    • and the capacity W (a nonnegative integer)
  • Output:
    • A subset $S \subset {1, 2,…, n}$ that maximizes $\sum_{i\in S} v_i $ subject to $\sum_{i\in S} w_i \leq W$

2. Analysis

  • Step 1: Formulate recurrence (optimal solution as function of solutions to “smaller subproblems”) based on a structure of an optimal solution
    • Let $S$ = a max-value solution to an instance of knapsack
      • case 1: suppose item $n\notin S$ => $S$ must be optimal with the first $n-1$ items (If $S^*$ were better than $S$ with respect to 1st $n − 1$ items, then this equally true w.r.t. all $n$ items - contradiction)
      • case 2: suppose item $n\in S$ => $S - {n}$ must be an optimal solution with respect to the 1st $n-1$ items and capacity $W-w_n$ (If $S^∗$ has higher value than $S − {n}$ && $total Size ≤ W − w_n$ ,then $S^∗∪ {n}$ has $size ≤ W$ and value more than $S$ - contradiction)
  • Step 2: Identify the subproblems
    • All possible prefixes of items ${1, 2, . . . , i}$
    • All possible (integral) residual capacities $x ∈ {0, 1, 2, . . . , W }$

  • Step 3: Use recurrence from Step 1 to systematically solve all problems.

Dijkstra’s algorithm

原理:

使用贪心算法不断更新sourse point到某些个point的最短距离，最终计算出到达所需要的destination的最短距离。或者通过一整个循环求出到每个点的最短距离。

requirements:

所有edge的长度是non-negative

setup:

  • number of vertex: n
  • number of edges: m
  • sourse vertex: s
  • set of vertex: V
  • set of vertex finished: X （当前已经算出最短路径的点集合）
  • 所有的最短路径： A
  • frontier: 当前所有X中指向V-X的edge集合

greedy objective function:

  • score 1: (regular)

    for e : frontier 

    score1[e] = A[e1] + length[e];

  • score 2: (for heap implementation)

    for e : frontier
    score2[e2] = min(A[e1] + length[e]);

步骤:

使用heap：复杂度 O(mlogn)

  1. 维护一个hashTable X, 维护一个heap H (V-X，key 是该点的score2)
  2. s放入X, A[s] = 0
  3. V-s 放入 H
  4. main while loop
    while (!H.empty()) {
    取出H最顶上的点 v 1）
    A[v] = score2[v];
    // 更新v所有邻接点 v’ 的score2:
    for e(v, v’) : E 2）
    H.delete(v’)
    score2[v’] = min(score2[v’], A[v] + length[e]);
    H.insert(v’)
    }

每次循环中，1）和 2）都要进行至少一次heap插入或删除操作所以时间复杂度 ~O((m+n)log n), 一般m>>n 所以 ~O(mlogn)

Minimum Spanning Tree

Posted on 2016-12-23 | In Algorithms

Minimum spanning tree

1. Definition

For undirected graph G = (V, E) (connected) and a cost ce for each edge e:E
the MST is the minimum cost tree T in E that spans all vertices. cost: sum of edge costs.

  • T has no cycles
  • The subgraph T is connected

2. Prim’s MST algorithm

Principle:

X: The set of vertices already in the minimum spanning tree

T: Current MST.

V - X: vertices’s not explored yet.

  • Choose any s in V, put s in X.
  • while X != V:
    • for for every e(w,v): w in X, v in V - X, choose the least cost one e’
    • put e’ in T, put v in X

Heap implementation:

  • Use heap to represent set V - X.
  • Set key to be the lowest cost connected with any vertex in V.
  • When v extracted from top, upgrade keys of some elements:
    • for every edge e(v, w) if w in V - X:
      • delete w from heap;
      • key(w) = min(key(w), cost(e(v, w)));
      • add new w to the heap;
  • time complexity: ~ m * log(n) m: # edges, n: # vertices

3. Kruskal’s MST algorithm

Principle:

  • T : the minimum spanning tree, initially an empty set
  • Sort all of the edges based on cost, in the edge set E
  • For each edge e in E with currently least cost, examine whether it cause a cycle in T (judging if there is a path from one to another by BFS/DFS)
  • If not, put e and its end points in T, remove e in E
  • Time complexity: ~m*n

Union-find implementation

  • Use a priority_queue to represent E
  • Every time when an edge e popped from the pq, we examine that in the union-find data structure, whether the two end points of e are connected. (~log(n))
  • If not, union two end point. (~log(n))
  • Time complexity: ~m*log(n)

4. Notes

  • If all edge costs are distinct, it turns out that MST is unique.
  • Cut Property: e is an edge of G. Suppose there is a cut (A, B) such that e is the cheapest edge of G that cross it. Then e belongs to the MST of G.

SCC

Computing the strongly connected components

Setup

A strongly connected component is a set of vertices and edges so that every pair of vertices in the set can get to each other through some edges in the set.

Given directed graph G = (V, E), finding all of the strongly connected components.

Kosaraju’s Two-Pass Algorithm

Gr = (V, Er), where Er is the set of edges in which every edge are the reverse version of the corresponding edge in E.

leader[] is an array that leader[i] is the leader of the strongly connected components which ith vertex belongs to.

The algorithm:

  • global variable s = NULL; stack finish_order;
  • for every vertex v:
    • if v was explored, continue;
    • DFS(Gr, v)
  • while (!finish_order.isEmpty())
    • v = finish_order.pop();
    • if v was explored, continue;
    • s = v;
    • DFS(G, v)

DFS(G, v):

  • mark v as explored;
  • set leader[v] = s;
  • for each edge e(v, w) of v:
    • if w was explored, continue;
    • DFS(G, w)
  • finish_order.push(v);

Notes

  • The directed graph can be seen as a “meta-graph”, which means that we can see each strongly connected component as a single vertex. If there are several SCCs (# > 1) and the original graph is connected, the “meta-graph” is acyclic.

  • The finishing order of a DFS-Loop:

    In every strongly connected component, there is at least one node finishing after all of the nodes in the component which is “after” the current component shown in the “meta-graph”

  • The above property guarantees that in the second DFS_Loop, we can always begin our traversal in the “front” SCC.
